Built-in features for production

Multi-Transport

Console, file rotation, PostgreSQL, MySQL, MongoDB, SQLite, Datadog, Segment, Mixpanel in one config.

NestJS Module

Drop-in LogixiaModule.forRoot(), works with DI, request context interceptors, and provider injection.

Request Tracing

AsyncLocalStorage trace IDs flow automatically across async boundaries. No manual propagation needed.

Built-in Log Search

NLP-powered pattern recognition, fuzzy search, and full-text indexing across all transports.

Log Redaction

Automatically scrub passwords, tokens, API keys, and PII before writing to any transport.

OpenTelemetry

Zero-dep W3C trace context support. Ships spans to any OTLP collector out-of-box.

Built for every use case 

import { LogixiaLogger } from 'logixia';

const logger = new LogixiaLogger({
  level: 'debug',
  transports: ['console', 'file'],
});

logger.info('Server started', { port: 3000 });
logger.error('DB connection failed', { host: 'localhost' });
// app.module.ts
import { LogixiaModule } from 'logixia';

@Module({
  imports: [
    LogixiaModule.forRoot({
      level: 'info',
      transports: ['console', 'file', 'postgresql'],
    }),
  ],
})
export class AppModule {}

// my.service.ts
constructor(private readonly logger: LogixiaLogger) {}

this.logger.info('User created', { userId });
const logger = new LogixiaLogger({
  transports: [
    { type: 'console', colorize: true },
    { type: 'file', filename: 'app.log', rotate: '1d' },
    {
      type: 'postgresql',
      connectionString: process.env.DATABASE_URL
    },
    {
      type: 'datadog',
      apiKey: process.env.DD_API_KEY
    },
    {
      type: 'segment',
      writeKey: process.env.SEGMENT_KEY
    },
  ],
});
// Trace IDs flow automatically
// No manual passing needed
app.use(logixia.tracingMiddleware());

// Anywhere in your async call chain:
logger.info('Processing order', { orderId: '123' });
